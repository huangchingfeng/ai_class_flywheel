## 單元詳細內容

### 【0:00-0:15】開場：為什麼業務需要寫內容？

**教學重點**
- 業務的競爭已經不只是產品，而是「信任」
- 客戶在找你之前，會先 Google 你、看你的社群
- 數據：有經營內容的業務，諮詢量平均多 3 倍

**案例分享**
> 小美的故事：從月成交 2 件到 8 件，只因為開始寫專業文章

**金句**
> 「你不寫，客戶就去找有寫的人。」

---

### 【0:15-0:35】觀念篇：AI 寫作的正確觀念

**常見誤區**
| ❌ 錯誤做法 | ✅ 正確做法 |
|------------|-----------|
| 直接叫 AI「幫我寫一篇文章」 | 先給角色、情境、期望 |
| 拿到就發，不修改 | AI 產初稿，你加觀點和案例 |
| 每次都從頭問 | 建立模板，重複使用 |

**黃金三角公式**

```
角色 + 情境 + 期望 = 好內容
```

| 元素 | 說明 | 範例 |
|------|------|------|
| **角色** | 告訴 AI 它是誰 | 「你是一位保險業務專家」 |
| **情境** | 提供背景脈絡 | 「我的客戶是 30-40 歲的小資族」 |
| **期望** | 明確說明產出 | 「寫一篇 500 字的專業文章」 |

**AI 寫作流程**
```
確定目的 → 給黃金三角 → AI 產初稿 → 加入你的觀點 → 調整風格 → 發布
```

### 【0:35-1:05】實作 1：專業文章

**目標**：用 AI 寫出一篇 500 字的專業文章

**步驟**
1. 選定主題（跟你的專業相關）
2. 填入黃金三角模板
3. 讓 AI 產出初稿
4. 加入你的案例和觀點
5. 調整風格，讓它「像你」

**提示詞模板：專業知識文章**
```
你是一位 [你的產業] 領域的資深專家，擁有 [X] 年實戰經驗。

請幫我撰寫一篇關於「[主題]」的專業文章。

目標讀者：[你的客戶類型]
文章目的：建立專業形象，讓讀者想進一步諮詢

文章結構：
1. 開頭：用一個真實案例或數據吸引注意
2. 問題：點出讀者常見的 3 個痛點
3. 解方：提供具體可執行的建議
4. 結尾：呼籲行動（歡迎私訊諮詢）

字數：500 字
語氣：專業但親切，像朋友在聊天，不要有 AI 味

寫作規則：
- 句子短，每段不超過 3 行
- 一定要有案例或數字
- 結尾 CTA 不硬賣，引導私訊領資料
```

**業務場景範例**

| 產業 | 主題範例 |
|------|----------|
| 保險 | 「30 歲前一定要懂的 3 個保險觀念」 |
| 房仲 | 「首購族最常犯的 5 個錯誤」 |
| 財顧 | 「為什麼你的投資總是賠錢？」 |
| B2B | 「選供應商前必問的 3 個問題」 |

**學員實作時間**：20 分鐘

**提示詞模板：案例見證貼文**

```
我提供的服務：[你做了什麼]

貼文要求：
- 開頭用成果數據吸引注意
- 中間說明客戶的改變
- 引用客戶原話增加可信度
- 結尾帶出服務價值，但不要太銷售感
- 150-200 字
```

### 【0:15-0:40】NotebookLM：建立你的專屬知識庫

**NotebookLM 是什麼？**
> 「它只會用你給它的資料回答，所以不會亂編。」

**與 ChatGPT 的差異**

| NotebookLM | ChatGPT |
|------------|---------|
| 只用你上傳的資料 | 會用網路知識（可能會錯） |
| 不會亂編 | 可能會「創作」內容 |
| 回答一定有根據 | 需要自己驗證 |

**業務的 5 大應用場景**

| 場景 | 做法 |
|------|------|
| **產品學習** | 上傳產品手冊，問「這個產品的 3 大賣點？」 |
| **客戶分析** | 上傳客戶資料，問「這家公司的痛點是什麼？」 |
| **競品比較** | 上傳競品資料，問「我們跟他們的差異？」 |
| **會議準備** | 上傳相關資料，產生背景摘要 |
| **新人訓練** | 上傳 SOP，產生測驗題和學習卡 |

**實作：建立你的產品知識庫**

**步驟**
1. 前往 notebooklm.google.com
2. 建立新筆記本，命名為「我的產品知識庫」
3. 上傳：產品手冊、DM、FAQ、培訓資料
4. 開始問問題

**提示詞範例**
```
請根據這些資料回答：

1. 我們產品的 3 大核心優勢是什麼？
2. 跟競品比，我們最大的差異化在哪？
3. 客戶最常問的問題有哪些？怎麼回答？
4. 什麼類型的客戶最適合這個產品？
```

**九大功能按鈕（業務最實用的）**

| 功能 | 業務怎麼用 |
|------|-----------|
| **摘要** | 快速了解一份文件的重點 |
| **測驗** | 自我測試產品知識 |
| **學習卡** | 通勤時複習 |
| **Podcast** | 開車時聽，邊聽邊學 |

**學員實作時間**：20 分鐘

### 【0:40-1:10】Perplexity：即時搜尋有來源的答案

**Perplexity 是什麼？**
> 「每一句話都有來源，點進去就能看原文。」

**為什麼業務要用 Perplexity？**
- 客戶問你問題，你要有根據的答案
- 回答時可以說「根據 [來源]，...」更有說服力
- 即時搜尋，資料最新

**5 個專業頁籤**

| 頁籤 | 功能 | 業務怎麼用 |
|------|------|-----------|
| **Finance** | 股票、財報 | 研究客戶公司財務狀況 |
| **Academic** | 學術論文 | 找專業數據佐證 |
| **Social** | Reddit、論壇 | 了解使用者真實評價 |
| **News** | 新聞 | 追蹤產業動態 |
| **Travel** | 旅遊 | 出差規劃 |

**實作：客戶背景快速研究**

**場景**：明天要拜訪一家新客戶，你需要快速了解他們

**提示詞**
```
請幫我研究 [客戶公司名稱]：

1. 公司基本資料（成立時間、員工數、營收規模）
2. 主要業務和產品
3. 近期新聞或動態
4. 可能面臨的挑戰或痛點

請用繁體中文回答，並標註資料來源。
```

**進階用法：Spaces 知識庫**
- 可以上傳自己的檔案
- AI 會結合你的檔案 + 網路搜尋結果
- 適合：產品資料 + 市場研究

**學員實作時間**：20 分鐘

### 【1:10-1:40】Gemini Deep Research：自動產出研究報告

**Deep Research 是什麼？**
> 「它會自己規劃研究計畫、自己找 50+ 來源、自己整理報告。」

**使用步驟**
1. 輸入研究主題
2. Gemini 自動產生研究計畫（可修改）
3. 點擊「開始研究」
4. 等待 3-5 分鐘
5. 取得完整報告（含所有來源）

**業務的應用場景**

| 場景 | 研究主題範例 |
|------|-------------|
| **市場調查** | 「2025 台灣 [產業] 市場趨勢」 |
| **競品分析** | 「[競品公司] 的產品策略分析」 |
| **客戶研究** | 「[客戶公司] 的數位轉型現況」 |
| **產業報告** | 「[產業] 未來 3 年發展預測」 |

**實作：產出一份競品分析報告**

**提示詞**
```
請針對「[你的產業] 前三大競爭對手」進行深度研究，產出分析報告。

報告需包含：
1. 三家公司的基本介紹
2. 產品/服務比較表
3. 定價策略分析
4. 各自的優勢和弱點
5. 市場機會分析
6. 我們的因應策略建議

字數：1500-2000 字
```

**進階輸出**
- 轉成 Podcast：開車時聽
- 轉成資訊圖表：發社群用
- 匯出 Google Doc：正式報告

**學員實作時間**：25 分鐘
