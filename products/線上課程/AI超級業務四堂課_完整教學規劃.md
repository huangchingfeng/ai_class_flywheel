## 【開場震撼】5 分鐘

### 真實案例對比

**先講故事，不講道理：**

```
「我認識兩個業務，都在賣同一個產品給同一家公司。

業務 A：
- 花了三天研究客戶，做了 30 頁簡報
- 見面時客戶問了一個他沒想到的問題
- 他說「我回去查查」
- 之後客戶沒再回他

業務 B：
- 用 AI 花了 30 分鐘研究客戶
- 他知道這家公司上個月剛換了採購主管
- 他知道這家公司正在做數位轉型
- 他知道競爭對手上週才被他們淘汰的原因
- 見面時客戶問什麼，他都答得出來
- 當場簽約

差別在哪？不是能力，是工具。」
```

### 震撼數據

```
傳統業務 vs AI 業務：
├── 客戶研究時間：3 天 vs 30 分鐘
├── 資訊完整度：60% vs 95%
├── 客戶驚豔指數：「還好」 vs 「你怎麼知道這個？」
└── 成交率：業界平均 vs 高出 3 倍
```

## 【觀念建立】10 分鐘

### 40 年經驗的第一個真相

**「客戶不是被說服的，是被理解的」**

```
❌ 傳統思維：我要說服客戶買我的產品
✓ 正確思維：我要讓客戶覺得「你最懂我的問題」

當客戶覺得你懂他，成交就是自然而然的事。
```

### 為什麼「準備」是最被低估的技能？

**業務成功公式：**

```
成交率 = 準備深度 × 溝通能力 × 持續跟進

大多數人只練溝通，卻忽略了佔 80% 權重的「準備」
```

### 傳統準備 vs AI 準備

| 項目 | 傳統方式 | AI 方式 |
|------|---------|--------|
| 公司基本資料 | 官網 + 新聞搜尋（1 小時）| 一句話指令（3 分鐘）|
| 競爭對手分析 | 一家一家查（3 小時）| Deep Research（10 分鐘）|
| 產業趨勢 | 讀報告（半天）| AI 整理重點（5 分鐘）|
| 決策者背景 | LinkedIn 爬文（1 小時）| AI 人物分析（5 分鐘）|
| **總計** | **8-12 小時** | **30-45 分鐘** |

**金句：「不是你不夠努力，是你的方法已經過時了」**

## 【觀念建立】10 分鐘

### 專業形象的三個層次

```
Level 1：答得出來
├── 客戶問什麼，你都能給答案
├── 不會說「我回去查查」
└── 建立「這個人很專業」的第一印象

Level 2：答得比客戶預期更好
├── 客戶問 A，你能延伸到 B 和 C
├── 讓客戶覺得「問對人了」
└── 建立「這個人真的懂」的信任

Level 3：問出客戶沒想到的問題
├── 你問的問題讓客戶說「對耶，我沒想過這個」
├── 從回答者變成引導者
└── 建立「這個人是專家」的權威

今天教你達到 Level 1 和 Level 2。
Level 3 是正課會深入教的。
```

### 為什麼記憶力不等於專業度？

**錯誤觀念：**
```
❌ 專業 = 記住所有產品規格
❌ 專業 = 背下所有 FAQ
❌ 專業 = 腦中有所有客戶資料
```

**正確觀念：**
```
✓ 專業 = 3 秒內找到正確答案的能力
✓ 專業 = 把對的資訊在對的時機給客戶
✓ 專業 = 讓客戶覺得「你隨時都準備好了」
```

**金句：「記憶力有限，系統力無限」**

### 跟進的三個誤解

```
誤解 1：「客戶會主動聯絡我」

誤解 2：「太常聯絡會很煩」
真相 → 不聯絡才會被忘記，適當的關心是加分

誤解 3：「跟進就是問『考慮得怎麼樣』」
真相 → 好的跟進是提供價值，不是追問結果
```

### 跟進的黃金時間表

```
【成交機率 vs 跟進頻率】

見面後 24 小時內跟進：成交率 + 40%
見面後 1 週內跟進：成交率 + 25%
見面後 1 個月後跟進：成交率 - 60%

最佳跟進節奏：
Day 1：感謝 + 會議重點整理
Day 7：提供承諾的資料 + 關心
Day 14：產業相關新聞分享
Day 30：詢問進度 + 新的價值
```

```
「我知道要跟進，但是...」

❌ 太忙了，見完這個要見下一個
❌ 客戶太多，記不住誰要什麼時候跟進
❌ 不知道要說什麼，總不能一直問『怎麼樣了』
❌ 寫 Email/訊息很花時間

這些問題，AI 都能解決。

```
大多數業務的問題：

- 花時間在不會買的人身上
- 好不容易成交，客戶用得不滿意
- 沒有轉介紹，永遠在找新客戶

先定義「最適合你的客戶畫像」，只找這種人
```

### Part 2：AI 幫你找到「正在需要」的客戶（10 分鐘）

**核心工具：Grok Deep Search + Gemini**

**什麼是「需求信號」？**

```
公司不會掛個牌子說「我們要採購」，
但他們會釋放信號：

💡 招聘信號
   → 「招聘 IT 主管」= 可能要做數位專案

💡 新聞信號
   → 「獲得融資」= 有預算做事了

💡 人事異動信號
   → 「新任營運長」= 新官上任要有新政策

💡 競爭信號
   → 「競爭對手導入 XX」= 他們會有壓力

💡 法規信號
   → 「新法規即將上路」= 必須做調整
```

**實際示範（Live Demo）：**

```
【找正在需要的公司】

指令（用 Grok）：
「搜尋過去 30 天內，台灣製造業有以下動態的公司：
1. 宣布數位轉型計畫
2. 招聘 IT、數據、數位相關職位
3. 獲得政府數位補助
4. 高層在公開場合談到效率或轉型

列出 10 家公司，每家附上：
- 公司名稱
- 相關新聞/動態
- 為什麼這是個好時機接觸
- 建議的切入話題」
```

**AI 產出範例：**

```
1. XX 科技（電子製造）

   切入：「看到 AA 公司做了 XX，你們有考慮嗎？」

（...繼續列出 7 家）
